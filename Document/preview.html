TÀI LIỆU THIẾT KẾ HỆ THỐNG FILE SERVER & PERMISSION SYSTEM

Mục tiêu xây dựng hệ thống quản lý file sử dụng Spring Boot, hỗ trợ upload file, folder tree, permission động, quota dung lượng, MinIO storage và audit log.

3. Cơ Chế Login Tạm Thời

Hệ thống tạm thời KHÔNG sử dụng JWT Authentication. Để đơn giản hóa việc development và test API, hệ thống sử dụng 1 biến static để lưu user hiện tại.

3.1 Nguyên Lý Hoạt Động


User gọi API login
        ↓
Truyền username
        ↓
Server lưu username vào biến static
        ↓
Mọi API tiếp theo sẽ sử dụng user này

3.2 Login API


POST /api/auth/login

Request Body


{
    "username": "admin"
}

Ví dụ Class Static


public class LoginContext {

    public static String CURRENT_USER;
}

Ví dụ Login Service


public void login(String username) {

    LoginContext.CURRENT_USER = username;
}

Ví dụ Lấy User Hiện Tại


String currentUser = LoginContext.CURRENT_USER;

Lưu Ý

Cơ chế này chỉ dùng cho:
  • Demo
  • Development
  • Testing local
Không phù hợp cho production vì:
  • Không hỗ trợ multi-user thật
  • Không thread-safe
  • Không bảo mật
  • Không hỗ trợ session/token

9. Quota System

Mỗi user có giới hạn dung lượng upload. Mặc định: 10GB / user

9.1 Flow


Upload File
    ↓
Lấy file size
    ↓
Kiểm tra used_storage + file_size
    ↓
Nếu vượt quota → reject
    ↓
Cho phép upload

9.2 Ví dụ

User Used Quota Upload Kết quả
A 9GB 10GB 500MB Success
B 9.8GB 10GB 500MB Reject
